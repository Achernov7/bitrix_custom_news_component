# Компонент «Новости» для 1С-Битрикс

Кастомный компонент вывода новостей с фильтрацией, пагинацией, детальной страницей и поддержкой SEF.

## Требования

- PHP 8.2 (Docker-образ: `php:8.2-fpm-bookworm`)
- 1С-Битрикс: Управление сайтом **26.150.0**, модуль `iblock`. Скачал с сайта архив "Управление сайтом. Стандарт"
- MySQL 8.0, Nginx 1.27
- Redis 7 (бэкенд кеша; PHP-расширение `phpredis`)

> Всё окружение собирается через Docker — устанавливать зависимости вручную не нужно.

## Развёртывание

```bash
docker compose up -d
```

Сайт: `http://localhost:8080/news/`  
Админка: `http://localhost:8080/bitrix/admin/` — логин `admin1`, пароль `admin1`

При первом старте контейнер автоматически восстанавливает Битрикс из дампа, устанавливает зависимости и создаёт 300 тестовых новостей.

### Сидер

```bash
# Заполнить 300 новостей (идемпотентно — повторный запуск пропускает существующие)
docker compose exec php php local/components/company/news_seed/bin/seed.php
```

Сидер создаёт 300 элементов в 5 разделах с датами из 5 лет (2020–2024), каждый третий — с превью-картинкой.

## Описание решений

### Архитектура

Компонент разделён на слои: `Repository` (запросы к БД), `Service` (бизнес-логика и форматирование), `Action` (логика страницы: `ListAction` / `DetailAction`), `Filter` (валидация GET). `component.php` только бутстрапит зависимости и делегирует нужному Action.

### Хранение данных

Инфоблок и элементы хранятся в штатных таблицах Битрикс. Официальный API (`CIBlockElement`, `CIBlockSection`) даёт совместимость с админкой, готовую фильтрацию, пагинацию и корректный сброс кеша через теги.

### SEF и маршрутизация

`CComponentEngine` — стандартный механизм Битрикс для SEF. Единственное правило в `urlrewrite.php` направляет `/news/*` в `news/index.php`; дальнейшую маршрутизацию (список / категория / деталь) берёт компонент. Страница категории — тот же список с разделом из URL-пути; GET-фильтры комбинируются с категорией по AND, пагинация сохраняет SEF-путь.

### Кеширование

Ключ кеша — `md5` от ID инфоблока, условий фильтра, номера страницы, размера страницы(если есть) и групп пользователя. Бэкенд — Redis: задаётся в `bitrix/.settings.php`, расширение `phpredis` ставится в `Dockerfile`, сервис — в `docker-compose.yml`. Компонент работает через стандартный `CPHPCache` и не зависит от конкретного движка.

> **?** `getSections()` и `locate()` выполняются на каждом запросе вне кеша компонента (2 запроса при cache hit). Стоит ли кешировать их? В целом самые частые, но легкие.

### Пагинация

`GetPageNavStringEx()` → компонент `bitrix:system.pagenavigation`. Ссылки страниц строятся от текущего URL и `$_GET` за вычетом `PAGEN_*` — GET-фильтры и SEF-путь сохраняются в пагинации автоматически.

### Экранирование

Репозиторий использует `GetNext()` — стандартный подход Битрикс: каждое поле возвращается в двух вариантах — `NAME` (прошло через `htmlspecialcharsEx`, выводится в шаблоне напрямую) и `~NAME` (сырое — для `SetTitle`, `AddChainItem`, обрезки анонса).

### Фильтрация

Год и диапазон дат транслируются в условия `>=DATE_ACTIVE_FROM` / `<=DATE_ACTIVE_FROM`. Диапазон имеет приоритет над годом. Категория задаётся GET-параметром или SEF-путём. При некорректном диапазоне (дата «с» > «по» или невалидный формат) выводится сообщение об ошибке, все условия фильтра сбрасываются и список показывается без фильтрации.
